# 0050 均價與損益計算邏輯分析報告

## 1. 均價不一致分析（Dashboard 102.48 vs APP 102.44）
* **券商 APP 均價為 102.44**：代表該筆交易實付手續費為 **40 元**（即 $(102.4 \times 1000 + 40) / 1000 = 102.44$）。
* **Dashboard 均價顯示 102.48**：是因為在系統/CSV 紀錄中，該筆買入交易的 `fee_tax` 欄位登錄為 **80 元**（即 $(102,400 + 80) / 1000 = 102.48$）。此落差是由於手續費輸入值不同，而非計算公式出錯。

## 2. 損益率不一致分析（Dashboard +4.17% vs APP +2.47%）
原系統在「庫存總覽」中的損益計算邏輯混雜了該檔個股的**歷史已結清交易紀錄**：
1. **已實現損益（前一次交易週期）**：`+10,619` 元（來自 7/13 買入、7/15 結清）。
2. **目前未實現損益（當前持有部位）**：假設現價為 `100.5` 元，估算未實現損益為 `-2,223` 元（已預扣賣出手續費與證交稅）。
3. **累計總損益**：`未實現損益 (-2,223) + 已實現損益 (+10,619) = +8,396` 元。
4. **歷史累計買入成本（分母）**：`第一週期買入 (92,631) + 當前持股買入 (102,480) = 195,111` 元。
5. **總報酬率**：`8,396 / 195,111 = 4.30%`（隨現價變動，當前呈現約為 **+4.17%**）。

相比之下，**證券 APP 的庫存損益**僅計算**當前持有部位**（即未實現損益與未實現報酬率，約為 +2.47%），不會納入歷史已結清的獲利與買入成本作為分母。因此，若要與 APP 顯示一致，庫存總覽表需修正為僅計算現有庫存。

## 3. 處理結論（2026-07-17）

* **分析 1 確認無誤**：引擎公式與券商一致（買入手續費計入成本）；102.48 來自該筆交易 `fee_tax` 登錄為 80。→ 以「交易紀錄 → 編輯」把該筆手續費改為 40 即可，均價將顯示 102.44。
* **分析 2 確認無誤**：原「累計總損益 / 總報酬率」刻意混入歷史週期（GAS 版同構設計）。→ 已修改 Dashboard：移除「已實現損益」「累計總損益」欄，「總報酬率」改為「未實現報酬率」＝未實現損益 ÷ 當前部位成本，完全對齊券商 APP 口徑；歷史已實現績效改由「年度收益」頁查看。
* **數字勘誤**：文中 APP 的「+2.47%」推測應為「-2.47%」——由 Dashboard +4.17% 反推現價約 100.2，當前部位必為虧損（約 -2.4%）；正負號更正後所有數字互相吻合。
* 注意：本系統未實現損益為「淨值」（預扣賣出手續費與證交稅），與部分券商 APP 的「毛損益率」可能有約 0.24% 的固定差異。
